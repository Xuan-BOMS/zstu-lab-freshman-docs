### 0.2 硬件部分

常用硬件主要有这些：

- 烧录器 / 调试器
   `ST-Link`、`J-Link`、`DAPLink`等（实验室有但容易丢失，可自行购买）。
- 开发板 / 核心板
  实验室有大量 `F1`、`F4` 开发板可直接使用，也可自行选择，51、audio、esp32等开发板不纳入学习计划。

- 基础外设模块（实验室有）

- 开发板用实验室现成的 `F1`、`F4` 就足够入门。
- 你自己选板子也可，入门学习阶段差别并不大。

- 知道什么是高低电平、上拉下拉、输入输出。
- 能看懂最简单的原理图连线。

## 1. STM32 基础开发

这一部分目的是完成基础任务。部分内容可先完成“实现”而不是“全部理解”。

建议顺序：

`GPIO -> 定时器 -> 中断 -> PWM`

### 1.1 任务一：GPIO 控制 LED、继电器，读取开关

这是 STM32 入门第一关。你要先理解最基本的GPIO输入和输出。

你需要完成：

- 操控GPIO电平，例如点亮一个 LED，并实现闪烁，或控制一个继电器吸合和断开。
- 读取一个开关或按键状态。

你需要理解的内容：

- 推挽输出、输入模式、上拉下拉分别是什么。
- 为什么有的按键按下是低电平，有的是高电平。

完成标准：

- 能重新配置一个 GPIO 输出口。
- 自己能解释电路是高电平有效还是低电平有效。

小问题：
- 什么是按键抖动，为什么要消抖。

### 1.2 任务二：认识定时器，用定时器改变 GPIO 状态

学完 GPIO 后，不要只会用 `HAL_Delay`。下一步要认识定时器。

你需要完成：

- 配一个基础定时器。
- 让它每隔固定时间翻转一次 GPIO。
- 了解 `软件延时` 和 `定时器定时` 两种方式。

你需要理解的内容：

- 定时器不是只用来“延时”，它本质上是一个计数器。
- 预分频、自动重装值决定了定时时间。
- 用定时器做周期任务，比在主循环里等更接近真实项目写法。

完成标准：

- 自己能根据时钟算出定时周期。
- 能修正 LED 闪烁周期。
- 能说清楚为什么阻塞式延时会影响程序同时做别的事。

小问题：
- 不同的定时器之间有什么区别，为什么相同配置下时间不同。
  
### 1.3 任务三：进一步认识定时器和中断，实现中断控制

中断是 STM32 开发中的核心概念。学到这里，你开始从“轮询思维”进入“事件思维”。

你需要完成：

- 配置一个定时器更新中断。
- 在中断回调里修改 GPIO 状态。
- 额外尝试一个外部中断按键，用按键触发 LED 状态切换。

你需要理解的内容：

- 什么叫中断触发，什么叫中断服务函数。
- 为什么中断里不要写太重的逻辑。
- 主循环和中断分别适合做什么事。

完成标准：

- 能解释“轮询”和“中断”的区别。
- 能定位“中断没进来”常见原因，例如没开 NVIC、回调写错、标志位没处理。

小问题：
- 中断优先级是什么，为什么有时候需要调整。

### 1.4 任务四：认识 PWM，实现波形控制

PWM 是后面控制电机、舵机、调光、驱动功率器件的基础。

你需要完成：

- 输出一路 PWM。
- 用 PWM 控制 LED 明暗变化，或者控制蜂鸣器、小电机转速。
- 修改频率和占空比，观察现象变化。

你需要理解的内容：

- PWM 本质上是定时器的一种输出方式。
- 频率和占空比分别影响什么。

完成标准：

- 能修改占空比从 10% 改到 90%，并看懂现象。
- 知道为什么电机控制和 LED 呼吸灯的 PWM 参数不会完全一样。
- 知道后面做控制时，PWM 往往就是“执行器输出”。

小问题：
- PWM 输出引脚和普通 GPIO 有什么区别，为什么有些引脚不能做 PWM 输出

## 2. 通讯相关

这一部分的目标是：学会一类总线，就能学会一类外设。

建议顺序：

`串口 -> I2C -> SPI -> CAN、USB 等其他总线`

### 2.1 任务五：串口

串口是最先要掌握的通信方式，也是最常用的调试方式，往往也是你和你的队友板子交流的方式。

你需要完成：

- 使用 `CH340` 或其他 USB 转串口模块连接开发板。
- 实现串口打印。
- 实现串口接收一个字符或一行命令，再控制 LED 或继电器动作。

推荐小任务：

- 上位机发送 `on`、`off`，控制 LED。
- 定时打印计数值或按键状态。

完成标准：

- 能独立配置串口波特率、收发引脚。
- 能排查串口乱码、收不到数据、收发反接等基础问题。
- 知道串口除了调试，还可以接很多简单模块。

小问题：
- 为什么串口通信有波特率，波特率不匹配会有什么现象。

### 2.2 任务六：I2C

I2C 常用于传感器、EEPROM、OLED 这类低速外设。

你需要完成：

- 接一个 `I2C` 外设。
- 能读出至少一个寄存器。
- 能把读取结果通过串口打印出来。

推荐外设：

- `OLED`
- `MPU6050` 这类惯性传感器

完成标准：

- 知道主机、从机、地址、读写寄存器在说什么。
- 遇到 I2C 无响应时，知道先查地址、上拉、电平、时序。
- 能理解模块的i2c例程，知道每个函数在干什么（虽然例程写的不一定好）。

小问题：
- I2C 的时序和电平要求为什么比较严格，为什么有时候需要加上拉电阻。

### 2.3 任务七：SPI

SPI 常用于速度更高的显示、存储、传感器等器件。

你需要完成：

- 接一个 `SPI` 外设。
- 完成至少一次读写。
- 用串口或显示器确认数据正确。

推荐外设：

- `TFT` 屏幕
- 高速 ADC 或部分陀螺仪模块

完成标准：

- 知道 `SCK`、`MOSI`、`MISO`、`CS` 分别干什么。
- 知道 SPI 有主从关系，也知道模式配置会影响通信是否成功。
- 知道为什么同样是“读寄存器”，SPI 和 I2C 的使用感受不一样。

小问题：
- SPI与I2C相比，为什么速度更快，但距离短。

## 3. 附加内容

### 3.1 RTOS

`RTOS` 不是新生必学内容，但可了解。

建议先完成前面的 GPIO、定时器、中断、PWM、串口、I2C、SPI，再开始看 `FreeRTOS`等操作系统。

完成标准：

- 能解释任务、调度、延时、队列这些最基本概念。
- 不会把 RTOS 当成“高级版 while(1)”。
- 知道用rtos会让代码更清晰，但也会增加复杂度。

小问题：
- 为什么有时候 RTOS 反而不适合做简单的控制任务，或者资源受限的项目。

## 4. 控制相关学习

我们做的控制大多较为简单，这部分学习也不难。
目标不是培养你去做复杂控制理论推导，而是让你具备做简单嵌入式任务的能力。
如果你在浙江的高中学过“通用技术”这门学科且还记得，那你会很快入手。（或者你在大学是控制类的？那你肯定比我了解的多）

你要先建立一个总认识：

- 控制系统通常有输入、测量、计算、输出四个环节。
- 嵌入式程序通常要把单纯的控制代码与具体实现分开写。
- 控制的具体结果会受很多因素影响，需要通过软件硬件等多方面去调试配合。

### 4.1 任务八：理解开环、闭环与采样周期


你需要知道：

- 开环：给多少输出，不看结果。
- 闭环：测结果，再根据误差继续调。
- 采样周期：控制程序不是连续运行的，而是按固定周期更新。

推荐小任务：

- 用按键切换 PWM 占空比，观察电机或风扇变化，这就是最简单的开环。
- 再引入一个测量量，例如编码器速度或传感器反馈，尝试按误差调节输出，这是闭环的起点。

完成标准：

- 能分清“命令值”“反馈值”“误差”。
- 知道控制程序为什么要固定周期执行。

小问题：
- 采样周期太长或太短会有什么问题，为什么有时候需要调整

### 4.2 任务九：状态机

状态机是嵌入式程序组织的基础，也是控制程序避免混乱的关键工具。

你需要知道：

- 系统不是只有“运行”一种状态，通常还有待机、启动、运行、异常、停止。
- 程序行为不只取决于输入，还取决于当前状态。
- 一旦项目开始变复杂，不用状态机，代码很容易乱。

推荐小任务：

- 写一个四状态系统：`idle -> ready -> run -> error`。
- 条件来自按键、定时器、传感器阈值。
- 错误状态下禁止电机输出，只允许复位后恢复。

完成标准：

- 能把“什么时候切状态”写清楚。
- 状态切换条件不靠一堆零散 `if` 硬拼。
- 出现异常时，程序行为是可预期的。

小问题：
- 为什么状态机更适合复杂系统。

### 4.3 任务十：PID

`PID` 是最常见的入门控制。先学会它，再去看别的控制方法会发现他真的很“入门”。

你至少要知道：

- `P`：误差大，输出就加大，作用直接。
- `I`：消除长期偏差，但太大容易积累过头。
- `D`：看变化趋势，帮助抑制震荡，但对噪声敏感。

入门时不要一上来就死记公式，更重要的是理解这三个量分别在“帮系统做什么”。

推荐小任务：

- 用一个带反馈的对象做速度控制或位置控制。
- 最推荐的是：`PWM 驱动电机 + 编码器测速 + PID 调速`。
- 如果暂时没有编码器，也可以先做一个简单模拟任务，例如让输出跟随目标值变化，先练程序框架和调参流程。

最基本的调试顺序：

- 先只用 `P`
- 再加 `I`
- 最后再决定要不要加 `D`

完成标准：

- 能解释超调、振荡、稳态误差这些现象。
- 能看串口打印的误差和输出变化。
- 能做最基本的参数调整，而不是只会抄一组参数。

### 4.4 任务十一：认识电机

做嵌入式控制，通常绕不开电机。

入门阶段至少要认识这些：

- 直流电机：最容易上手，适合学 PWM、方向控制、测速。
- 舵机：适合学固定周期 PWM 和角度控制。
- 步进电机：适合理解脉冲控制和位置控制。
- 无刷电机：后续很常见，但入门门槛更高。

你需要知道的不是电机分类背诵，而是下面这些问题：

- 这个电机怎么驱动。
- 需不需要驱动板。
- 能不能直接用单片机 IO 口带。
- 怎么测它的速度、位置、电流。

推荐小任务：

- 直流电机正反转控制。
- PWM 调速。
- 如果有编码器，再做一个闭环调速。

完成标准：

- 知道“单片机控制电机”和“单片机直接带电机”不是一回事。
- 知道驱动、电源、续流、保护这些词为什么重要。

### 4.5 任务十二：认识基础传感器

控制离不开传感器。入门阶段建议按“最常见、最能直接用于闭环”的顺序去认识。

先认识这几类：

- 按键、限位开关
- 编码器
- 电位器
- 电流传感器
- 距离传感器
- 惯性传感器：加速度计、陀螺仪、IMU

这里尤其要先分清：

- 加速度计主要测线加速度，也常用于静态姿态估计。
- 陀螺仪主要测角速度，动态响应快，但会漂移。
- IMU 往往是多种惯性传感器的组合。

推荐小任务：

- 读取一个 `IMU` 模块的原始数据。
- 通过串口输出角速度或加速度。
- 做一次静止零偏校准。
- 用最简单的低通滤波或互补滤波改善数据表现。

完成标准：

- 知道传感器原始值不是“拿来就能直接用”的。
- 知道为什么控制前常常要先做校准、滤波、限幅。

### 4.6 任务十三：把控制程序写得像一个系统

控制项目真正难的地方，经常不是公式，而是程序组织。

建议你尽早形成下面这个结构意识：

- 采集层：读传感器
- 估计层：滤波、换算、校准
- 控制层：状态机、PID、保护逻辑
- 执行层：PWM、方向、继电器
- 通信层：串口打印、上位机调参

推荐综合任务：

- 做一个“目标速度可调”的电机控制小系统。
- 按键或串口输入目标值。
- 编码器提供反馈。
- PID 负责调速。
- 状态机负责启动、停止、故障保护。
- 串口负责打印关键变量。

完成标准：

- 你的工程不是所有逻辑都塞在 `main.c` 里。
- 出现问题时，你知道是采集问题、控制问题还是执行问题。

## 5. 建议学习节奏

如果你是刚进实验室的新生，比较稳妥的顺序是：

### 第 1 阶段

- 跑通工具链
- 点灯
- 按键输入
- 串口打印

### 第 2 阶段

- 定时器
- 中断
- PWM
- I2C 或 SPI 接一个外设

### 第 3 阶段

- 读一个传感器
- 控一个执行器
- 写一个状态机

### 第 4 阶段

- 做一个最小闭环
- 再看 `FreeRTOS`

## 6. 最后建议

- 新人最怕的不是不会，而是总想一步到位。
- 嵌入式学习最有效的方法不是看很多，而是一个任务一个任务做完。
- 每学一个外设，都问自己三个问题：它是干什么的、它怎么调、它坏了怎么查。
- 真正拉开差距的，往往不是谁会背更多接口，而是谁更会定位问题、组织代码、把系统做完整。

如果你按这条路线走完，后面再去做比赛、小项目、传感器节点、电机控制、RTOS，都会顺很多。
